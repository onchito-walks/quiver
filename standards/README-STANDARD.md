# README Architecture Standard

Every project under onchito-walks opens with a human paragraph that answers:
what it is, why it exists, why it is needed, what it lives for.

## The opening paragraph
3-6 sentences. No badges, tables, or install instructions.
Must tell a story: problem → solution → why it matters.

## After the opening
Architecture-first, tech-stack-second.
Diagram → principles → performance → stack → install → limitations.

## Enforcement
- New repos: template applied before first commit
- Existing repos: audited against this standard
- Quiver nightly maintainer checks READMEs

See GBrain: standards/readme-architecture for the full standard.
